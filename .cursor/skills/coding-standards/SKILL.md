# YCS 開發標準

- **拒絕過度設計**：優先使用 Server Actions，避免複雜設計模式。
- **檔案長度約束**：單一檔案限制在 300 行以內，超長時必須主動提議拆分組件。
- **JSDoc 要求**：所有 Function 必須使用繁體中文註解業務邏輯。
- **不確定性原則**：若需求不明確，實作最直觀的解法並標示假設。
- **UI 反饋**：使用 shadcn/ui + Toast 顯示繁體中文執行結果。
- **歷史資料原則**：面對舊系統「一單多掛 (M:N)」的例外情況，應在匯入時進行資料清洗與拆分，保持資料庫「1:N」的純潔結構。
